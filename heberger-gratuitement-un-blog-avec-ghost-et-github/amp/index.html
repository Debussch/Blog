Hébergez votre blog gratuitement avec Ghost et GitHub!

by ThoDe

2017-11-30

Alors que WordPress est le mastodonte des systèmes de gestion de contenu sur le web, de sérieuses alternatives se développent chaque jour sur internet. Parmi elles, nous retrouvons Ghost CMS, propulsé en 2013 à la suite d'une campagne de crowndfounding. Ce projet open sources sous license MIT veut simplifier l’édition de posts pour les Bloggers. Développé en Javascript, vous découvrirez comment héberger gratuitement votre propre Ghost Blog grâce à GitHub Pages dans la suite de cet article.

Comment allons-nous procéder?

GitHub est LA plateforme favorite des développeurs qui peuvent grâce à elle gérer leurs versions facilement. Elle permet entre autre, grâce à son service GitHub Pages d'héberger gratuitement des pages web au format HTML. Cependant, elle n'offre pas la possibilité d'exécuter des scripts ou de gérer une base de donnée. Nous allons donc apprendre à héberger localement notre propre site avec NodeJs et npm avant d'en créer une copie statique. C'est cette dernière que nous déploierons ensuite sur GitHub grâce à quelques lignes de commandes.

Installation de l'environnement sous Windows

Prérequis

  1. Git et son Bash : découvrez la puissance du shell Unix
  2. NodeJs avec nvm : gérez facilement vos versions de NodeJs
  3. npm : importez des paquets NodeJs
  4. knex-migrator : installez votre propre base de donnée
  5. ghost-static : hackez Ghost CMS et générez des pages statiques

Installation de Git

Git est un système de gestion de version décentralisé créé par Linus Thorvald, le créateur de Linux. Il permet d'interagir avec la plateforme GitHub à partir de son Bash ou de la console de Windows (en fonction des paramètres choisis lors de l'installation). Téléchargez Git et suivez les étapes par défauts de son installateur.

Installation de NodeJs avec nvm

Nvm pour "Node Version Management" est un système de gestion de versions pour NodeJs. Il permet, entre autre, de pouvoir sauter d'une version a une autre en quelques secondes en cas de besoin. Téléchargez nvm puis suivez les étapes de l'installateur.

Ceci fait, lancez la console Windows et installez NodeJs en version 8.9.1 en exécutant la commande suivante :

    nvm install 8.9.1

Installation de npm : le gestionnaire de paquet de NodeJs

Une fois que NodeJs a terminé de s'installer, importez son gestionnaire de paquet à l'aide de la commande :

    node install npm

Installation de knex-migrator : votre gestionnaire de base de données

Procédez maintenant à l'installation du système de gestion de bases de données comme suit :

    npm install -g knex-migrator

Installation de Ghost-static

Grace à ce paquet NodeJs, vous pourrez générer des pages statiques de votre Ghost blog. Procédez a son installation avec la ligne de commande suivante :

    npm install -g ghost-static

Ça y est, votre ordinateur est maintenant équipé de tous les outils nécessaires a l'installation et l'exécution de votre blog Ghost en local. Plus que quelques étapes avant de le voir apparaitre dans votre navigateur !

Création du projet

Téléchargez d'abord la dernière version de Ghost en vous rendant sur le site de la Fondation.

Créez ensuite un dossier dev à la racine de votre disque dur et décompressez le contenu de l'archive à cet endroit.

Retournez sur l'invite de commandes de Windows et rendez-vous dans le dossier dev. Par défaut, lorsque vous ouvrez l'invite de commandes, il se situe au niveau de votre dossier utilisateur. Saisissez donc la commande suivante pour arriver directement dans le dossier :

    cd ../../dev

Importez les modules NodeJs dans votre dossier en tapant :

    npm install

Ceci fait, installez votre base de données avec :

    knex-migrator init

Ça y est, votre projet est enfin prêt a être exécuté !

Lancez votre blog en local

Nous y sommes enfin, pour exécuter Ghost en local il vous suffit de vous rendre dans votre dossier dev et de saisir la ligne de commande suivante :

    npm start

Votre blog est maintenant accessible à l'adresse localhost:2368.

Lors de la première exécution, pensez à créer votre compte administrateur en vous rendant sur localhost:2368/ghost/ et suivez les instructions a l'écran.

Vous accèderez ensuite au panneau d'administration de votre nouveau blog.

Passons maintenant à l'exportation vers Github.

Exportez votre Ghost blog vers GitHub Pages

Si ce n'est pas déjà fait, rendez-vous sur GitHub afin de créer votre compte utilisateur.

Créez ensuite un nouveau repository public en cliquant sur le + en haut a droite de l'interface.

Retournez ensuite dans votre dossier dev et créez un fichier ghost-static.json à la racine de votre projet. Puis, collez-y les lignes qui suivent :

    {
        "ghostURL": "https://debussch.github.io/blog",
        "tmpFolder": "tmp/",
        "staticFolder": "blog/",
        "publicURL": "https://NOM_DE_COMPTE.github.io/NOM_DU_REPO"
    }

Pensez à remplacer la dernière ligne par les informations saisies précédemment sur GitHub.

Retournez ensuite sur l'invite de commande et exécutez la commande qui suit afin de générer une copie statique de votre blog dans le dossier /blog :

    ghost-static

Ouvrez une fenêtre Git Bash et déplacez-vous dans le dossier blog :

    cd ../../dev/blog

Collez les lignes de commande suivantes en prenant soin de remplacer les variables de la cinquième ligne :

    echo "# Mon Ghost Blog" >> README.md
    git init
    git add README.md
    git commit -m "first commit"
    git remote add origin git@github.com:NOM_DE_COMPTE/NOM_DU_REPO.git
    git push -u origin master
    git checkout -b gh-pages
    git add .
    git commit -m "Initial commit"
    git push origin gh-pages

Félicitations, votre blog est enfin en ligne : https://NOM_DE_COMPTE.github.io !
